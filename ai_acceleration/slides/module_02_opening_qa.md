<!-- Slide 1: Opening Question -->
# What Takes Too Much Time?

<div style="font-size: 0.8em; margin-top: 2em;">
**You know your business better than I do.**

Before I show you what AI can do,
tell me: **what takes too much time in your day?**
</div>

<div style="margin-top: 2em; font-size: 0.7em; color: #888;">
Rules: Call it out | I'll write it fast | No wrong answers
</div>

<aside class="notes">
Time: 9:10-9:12 (2 min)

Purpose: Shift from instructor-led to participant-led, create ownership

Instructions:
- Ask the opening question directly
- Set up whiteboard/digital board with 3 columns: Job/Task | Time Spent | Pain Level (1-5)
- Set expectations: "We'll capture everything fast, no debate yet"
- Transition: "Let's fill this board..."
</aside>

  <div style="font-size: 0.7em; color: #ccc; margin-top: 0.5em;">
    Writing listing descriptions • Responding to same client questions • Creating social posts<br/>

<aside class="notes">
Time: 9:12-9:26 (14 min)

PROCESS:

1. RAPID-FIRE (7 min):
   - Call on participants or take volunteers
   - Each names 1-2 time-consuming tasks
   - Write quickly on board - exact wording doesn't matter
   - NO discussion yet, just capture
   - If room is quiet, use examples shown on slide

2. CLUSTERING (3 min):
   - Group related tasks together
   - Common clusters: Content creation, Client communication, Market research, Transaction coordination
   - Ask: "Did I miss any? Should these be combined?"

3. DOT VOTING (4 min):
   - Each participant gets 3 votes (real or virtual dots)
   - Vote for tasks they most want to solve with AI
   - Tally and circle top 5-7 jobs
   - These become "Target Jobs" for the day

Success criteria: 15-20 jobs identified, top 5-7 prioritized

Energy: Keep this high-energy, call and response, use humor when jobs are relatable
</aside>

<aside class="notes">
Time: 9:26-9:30 (4 min)

PURPOSE: Debrief and transition

Key takeaways to emphasize:
- Point out specific patterns you observed
- Acknowledge shared pain points (builds peer learning foundation)
- Build confidence: "AI is great at most of these"
- Create anticipation for solving these problems

Q&A prompts:
- "Surprised by what others said?"
- "Anyone have a job that didn't make the list but you're curious about?"

TRANSITION:
"Before we jump into tools and tactics, we need mental models - how to think about AI so you use it right and avoid mistakes. That's Module 3."

Common pitfalls to avoid:
- Don't let one person dominate - spread participation
- Don't debate feasibility yet - just capture
- Don't let this run beyond 20 min total

Learning Outcomes Met:
✓ Surface top priority use cases for the room
✓ Create shared understanding of pain points
✓ Identify 5-7 "jobs to be done" that AI can address
✓ Build peer learning foundation (realize others share challenges)
</aside>
