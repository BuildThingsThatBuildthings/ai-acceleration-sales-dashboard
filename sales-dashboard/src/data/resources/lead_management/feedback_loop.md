# Feedback Loop System

## Purpose

Every call teaches you something. This system ensures:
1. Learnings get captured
2. Patterns get identified
3. Scripts/objections get updated
4. Data quality improves over time

**The goal: Each week, you're better than the last.**

## Updating the System

### When to Update Objection Library

Add a new objection when:
- You hear it 3+ times in a week
- You didn't have a good response
- It's substantively different from existing objections

**New Objection Template:**
```
Objection: "[Exact words they use]"
What they mean: ____________________________________
Context: __________________________________________
Response that works: ________________________________
Response that doesn't work: __________________________
```

### When to Update Scripts

Update the script when:
- A phrase consistently gets better results
- A phrase consistently causes friction
- You discover a new opening/angle

**Script Update Template:**
```
Date: ____
Section: [ ] Opening [ ] Discovery [ ] Pitch [ ] Close [ ] Objection
Change: ___________________________________________
Reason: ___________________________________________
Result after change: __________________________________
```

### When to Update Value Props

Update value messaging when:
- You hear new pain points repeatedly
- A benefit resonates strongly
- A benefit falls flat

**Value Prop Update Template:**
```
Date: ____
Current message: ____________________________________
New message: ______________________________________
Why: _____________________________________________
Evidence: _________________________________________
```

## Team Knowledge Sharing

If multiple people are calling:

### Weekly Huddle Agenda (15 min)
1. **Wins** - What worked this week? (3 min each)
2. **Struggles** - Where are we stuck? (3 min each)
3. **Objections** - New ones? Better responses? (5 min)
4. **Data Issues** - Patterns in bad data? (2 min)
5. **Adjustments** - What we'll try next week (2 min)

### Shared Learning Log
```
Date: ____
From: [Team member name]
Learning: _________________________________________
Context: __________________________________________
Recommendation: ___________________________________
```

## Feedback Loop Quick Actions

### Immediate (During Call)
- [ ] Note exact objection language
- [ ] Flag data quality issues
- [ ] Record what worked/didn't

### Daily (End of Day)
- [ ] Review notes for patterns
- [ ] Update CRM with all calls
- [ ] Note tomorrow's adjustments

### Weekly (Friday)
- [ ] Analyze objection frequency
- [ ] Update scripts if needed
- [ ] Review data quality issues
- [ ] Share learnings with team

- Higher confidence on calls
